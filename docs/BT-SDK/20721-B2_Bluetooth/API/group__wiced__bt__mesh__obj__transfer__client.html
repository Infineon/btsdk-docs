Object Transfer Client Message Handler.

The Mesh Core library calls this function for each message received. The function returns WICED_TRUE if the message is destined for this specific model and successfully processed, and returns WICED_FALSE otherwise. Generic Level Server device. The function parses the message and if appropriate calls the parent back to perform functionality.

Parameters
p_eventMesh event with information about received message.
p_dataPointer to the data portion of the message
data_lenLength of the data in the message
Returns
WICED_TRUE if the message is for this company ID/Model/Element Index combination, WICED_FALSE otherwise.

Object Transfer Start.

A client model above object transfer client or an application can call this function to start actual Object transfer. The tranfer is done using the multicast to a group address. The object is split into blocks which is then split into chunks. The group list is used by the object transfer client to identify which chunks of a block has not been received and retransmit them.

Parameters
group_addrGroup address to multicast the chunks
p_group_listList of the nodes currently included in the group
group_sizeNumber of the elements in the group list
object_idObject ID for the current transmission
object_sizeSize of the object to distribute
block_size_logSize of each block
chunk_sizeSizeof of each chunk
app_key_idxApp Key Index of the key to be used for securing the data
Returns
WICED_TRUE if distribution has been started successfully
